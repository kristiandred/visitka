# Сайт-визитка с административной панелью

Современный сайт-визитка с административной панелью для управления контентом и загрузкой фотографий.

## Возможности

### Для посетителей
- Адаптивный дизайн для всех устройств
- Современный интерфейс с анимациями
- Галерея фотографий
- Контактная информация
- Социальные сети

### Для администратора
- Полное управление контентом сайта
- Загрузка фотографий с рабочего стола
- Управление категориями фотографий
- Редактирование текстового контента
- Изменение контактной информации
- Управление социальными ссылками

## Технологический стек

- **Backend**: Flask (Python)
- **База данных**: SQLite
- **Frontend**: HTML5, CSS3, JavaScript (Bootstrap 5)
- **Обработка изображений**: Pillow
- **API**: RESTful API

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <repository-url>
cd visitka
```

### 2. Создание виртуального окружения
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Запуск приложения
```bash
python app.py
```

Приложение будет доступно по адресу:
- **Публичный сайт**: http://localhost:5000
- **Административная панель**: http://localhost:5000/admin

## API эндпоинты

### Контент
- `GET /api/content` - Получить весь контент
- `GET /api/content/<key>` - Получить конкретный контент
- `POST /api/content` - Обновить контент

### Фотографии
- `GET /api/photos` - Получить все фотографии
- `POST /api/photos` - Загрузить новую фотографию
- `DELETE /api/photos/<id>` - Удалить фотографию
- `GET /uploads/<filename>` - Получить файл фотографии

## Управление контентом

### Через административную панель

1. Откройте http://localhost:5000/admin
2. Используйте боковое меню для навигации:
   - **Контент**: Редактирование текстовой информации
   - **Фотографии**: Управление галереей
   - **Настройки**: Системная информация

### Редактирование контента

Доступные поля для редактирования:
- `site_title` - Заголовок сайта
- `site_description` - Описание сайта
- `about_text` - Текст о себе
- `contact_email` - Email контакта
- `contact_phone` - Телефон контакта
- `social_links` - Социальные сети в формате JSON

### Социальные сети формат JSON
```json
{
    "github": "https://github.com/username",
    "linkedin": "https://linkedin.com/in/username",
    "facebook": "https://facebook.com/username",
    "twitter": "https://twitter.com/username",
    "instagram": "https://instagram.com/username"
}
```

## Загрузка фотографий

### Через административную панель
1. Перейдите в раздел "Фотографии"
2. Нажмите "Загрузить фото"
3. Выберите файл с компьютера
4. Добавьте описание и выберите категорию
5. Нажмите "Загрузить"

### Поддерживаемые форматы
- JPG/JPEG
- PNG
- GIF
- WebP

### Максимальный размер файла
- 16MB

## Категории фотографий
- `general` - Общее
- `portfolio` - Портфолио
- `personal` - Личное
- `work` - Работа

## Безопасность

- Загрузка файлов только с расширениями изображений
- Ограничение размера файлов
- Уникальные имена файлов для предотвращения коллизий
- Валидация данных на стороне сервера

## Резервное копирование

База данных SQLite хранится в файле `site_data.db`. Для создания резервной копии:

1. Остановите приложение
2. Скопируйте файл `site_data.db`
3. Скопируйте папку `static/uploads/` с фотографиями

## Разработка

### Добавление новых полей контента

1. Добавьте поле в модель `SiteContent` в `app.py`
2. Добавьте поле в форму в `admin.html`
3. Обновите JavaScript в `admin.js` для сохранения нового поля
4. Обновите JavaScript в `main.js` для отображения нового поля

### Кастомизация стилей

- Публичный сайт: `static/css/style.css`
- Админ панель: `static/css/admin.css`

### Добавление новых API эндпоинтов

Добавьте новые маршруты в `app.py` следуя существующему шаблону.

## Troubleshooting

### Проблема: База данных не создается
**Решение**: Убедитесь, что у приложения есть права на запись в директорию проекта.

### Проблема: Фотографии не загружаются
**Решение**: Проверьте права на запись в папку `static/uploads/`.

### Проблема: Сайт не доступен извне
**Решение**: Измените `host='0.0.0.0'` в `app.py` для доступа из сети.
